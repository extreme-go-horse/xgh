# for-against

Run a FOR/AGAINST design review -- parallel steel-man debate on a design decision.

## When it runs

- **On demand:** `/xgh-for-against`

## What it does

1. Takes a design proposal or approach
2. Dispatches two parallel subagents:
   - **FOR**: Steel-mans the strongest case in favor
   - **AGAINST**: Steel-mans the strongest case against
3. Synthesizes both perspectives into a balanced assessment
4. Highlights key risks, trade-offs, and recommendations

## When to use

- Before committing to a major architectural decision
- When evaluating competing approaches
- When you need to stress-test assumptions
